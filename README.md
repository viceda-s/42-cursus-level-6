## 🤝 42_collaborative_resume – Work in Progress 🚧

### Objective
Develop **communication, self-reflection, and professional writing** skills by conducting mutual interviews with a peer and collaboratively crafting each other's resumes — simulating a real-world recruitment scenario.

> 👥 Group project · Team of 2

### Context
Working as junior recruiters at **Technov**, a tech talent marketplace, each pair receives a client brief from *StartUpDigital Services* looking for junior freelance developers. The goal is to interview your partner and produce a polished, job-ready resume tailored to the client's needs.

### Key Features
- Mutual interviews covering personal background, education, experience, projects, and skills
- Resume writing in a standard format (Markdown, LaTeX, or DOCX), exported as PDF
- Peer exchange and constructive feedback loop
- Professional social media profile linked (LinkedIn preferred)

### Submission Structure
```
collaborative_resume/
├── resume_[login1].pdf
└── resume_[login2].pdf

media/
├── media_[login1].txt
└── media_[login2].txt
```

### Skills Being Developed
- Professional communication and active listening
- Self-assessment and articulating strengths clearly
- Resume writing tailored to a specific job market
- Organisation, adaptability, and interpersonal collaboration

## 🌐 ft_transcendence – Work in Progress 🚧

### Objective
Build a **full-stack web application** as a team — the content and direction are chosen by the group. The project must reach a minimum of **14 module points** through a combination of mandatory and optional modules, covering areas such as gaming, user management, DevOps, AI, cybersecurity, and more.

> 👥 Group project · Team of 4-5 

### Mandatory Requirements
- Full-stack web app: **frontend + backend + database**
- Single-command deployment via **Docker / Podman**
- **HTTPS** everywhere on the backend
- Basic **user authentication** (email + password, hashed & salted)
- **Multi-user support** with concurrent sessions and real-time updates
- Privacy Policy and Terms of Service pages
- Clean Git history with contributions from all team members
- A comprehensive **README.md** (team info, tech stack, DB schema, modules, contributions)

### Module System
14 points required to pass. Each **Major module = 2 pts**, each **Minor module = 1 pt**. Up to **5 bonus points** for additional modules beyond the minimum.

Available module categories: Web, Accessibility & i18n, User Management, Artificial Intelligence, Cybersecurity, Gaming & UX, DevOps, Data & Analytics, Blockchain, and Modules of choice.

### Team Roles (Required)
- **Product Owner (PO)** – product vision, backlog, feature validation
- **Project Manager / Scrum Master** – coordination, planning, risk management
- **Technical Lead / Architect** – architecture decisions, code quality
- **Developers** – feature implementation, testing, documentation

### Current Focus
- Deciding on the project concept and finalizing module selection
- Assigning team roles and setting up project management tooling
- Designing the application architecture and database schema
- Setting up the containerized development environment

### Challenges Faced
- Aligning the team on a shared project vision and module strategy
- Balancing technical ambition with realistic scope and timeline
- Designing a system that satisfies all general and technical requirements from the start
- Coordinating contributions across 4–5 developers on a long-running project

### Skills Being Developed
- Full-stack web development (frontend + backend + database)
- Team coordination, agile practices, and role-based responsibilities
- Real-world application design: security, scalability, accessibility
- Containerization and deployment with Docker
- Choosing and integrating modules across diverse technical domains
